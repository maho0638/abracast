# Abracast Research Playbook

Abracast does not recommend Abstract projects by hype alone.

Every project should pass a research and safety review before receiving a `Support` label.

## Step 1 - Source trail

Collect:

- Official website
- Portal or ecosystem listing
- Discord or docs link
- GitHub or contract source if available
- Explorer or marketplace link if relevant
- Independent directory reference such as Kura or Alchemy

If a project only has one directory listing and no official source trail, it should stay `Watch` or `Avoid`.

## Source confidence

Use this scale:

- `High`: official site/docs plus reliable ecosystem or third-party confirmation
- `Medium`: official source exists, but important safety, audit, or economic data is missing
- `Medium-low`: source trail is fragmented or conflicts with Abstract ecosystem fit
- `Low`: mostly directory-backed, needs official confirmation
- `None`: no reliable source trail or unsafe claim pattern

`Support` requires `High` source confidence.

## Step 2 - User loop

Answer:

- What does a user actually do?
- Why would they return next week?
- Is there a real game, tool, marketplace, or social loop?
- Does the project still make sense without XP?

## Step 3 - Economic logic

Answer:

- How can it create activity for Abstract?
- Does it create trades, swaps, mints, tips, marketplace usage, prize pools, or recurring gameplay?
- Is the economy sustainable or just reward extraction?
- Who pays, who earns, and why?

## Step 4 - Wallet and contract risk

Check:

- Does it ask for a seed phrase or private key?
- Does it use unclear approvals?
- Are contracts verified?
- Is there an audit, badge, open source repo, or community verification?
- Is the project DeFi, gambling, or prize-pool based?

High-risk categories need stronger proof before support.

## Step 5 - Streamer value

Answer:

- Can a creator explain it live?
- Can viewers understand the loop quickly?
- Is there skill, strategy, comparison, competition, or discovery?
- Would a stream bring real users to Abstract apps?

## Step 6 - Final label

- `Support`: useful, streamable, economically meaningful, and trusted enough
- `Watch`: promising but missing proof, polish, or trust evidence
- `Avoid`: weak value, unclear risk, fake claim pattern, or low trust

## Security rule

Security beats growth.

If a project looks exciting but the wallet flow, source trail, or economic risk is unclear, Abracast should not mark it `Support`.
