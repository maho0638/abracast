# Abracast Curation Governance

Abracast exists to route attention toward useful Abstract projects while protecting users from weak, risky, or under-verified apps.

## Core rule

Security beats growth.

No project receives `Support` unless all three are true:

- Strong ecosystem value
- Strong trust score
- High source confidence

## Decision process

### Support

Requires:

- High source confidence
- Official site or docs
- At least one additional reliable source
- Clear user loop
- Clear economic logic
- No unresolved wallet safety issue
- Risk notes documented
- Streamer mission documented

### Watch

Use when:

- Project is promising but under-verified
- Source trail is incomplete
- Audit or badge status is unclear
- Wallet flow needs review
- Economy is promising but unproven

### Avoid

Use when:

- No reliable source trail
- Private-key or seed-phrase pattern
- Misleading XP, reward, or claim language
- DeFi/chance/trading mechanics without safety proof
- Abstract ecosystem fit is unclear or misleading

## Score changes

Every score change should include:

- Date
- Project
- Old label
- New label
- Source links
- Why the change helps users
- Safety impact

## Weekly cycle

1. Review new Portal/Kura projects.
2. Add evidence links.
3. Update source confidence.
4. Recalculate Support / Watch / Avoid.
5. Publish Weekly Abstract Picks.
6. Open issues for missing evidence.
7. Ask Discord for corrections and feedback.

## Conflict rule

If a project has high value but unclear trust, trust wins.

Keep it `Watch` until evidence improves.
